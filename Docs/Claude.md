# **Ellams — Claude / AI Development Guide**

## **1\. Purpose**

This file guides AI-assisted development of the Ellams website.

The goal is to build a clean, editorial-style website with strong design consistency and minimal complexity.

---

## **2\. Core Principles**

### **2.1 Design First**

* Prioritize visual hierarchy over features  
* Keep UI minimal  
* Use large typography and spacing

---

### **2.2 No Feature Bloat**

* Do not add:  
  * likes  
  * comments  
  * profiles  
* This is NOT a social app

---

### **2.3 Consistency Over Creativity**

* Reuse components  
* Maintain consistent spacing, colors, and typography

---

## **3\. Tech Stack (Recommended)**

* Frontend: Next.js  
* Styling: Tailwind CSS  
* Content: Static JSON or simple CMS (Notion or local data)

---

## **4\. Component Structure**

### **Core Components**

* Navbar  
* SidebarMenu (slide-out)  
* HeroSection  
* ContentGrid  
* ContentCard  
* PostPage  
* Footer

---

## **5\. Styling Rules**

### **Colors**

* Primary: \#2B67A3  
* Background: \#F5F5F0  
* Text: \#111111

---

### **Typography**

* Use bold sans-serif fonts  
* Headings should be large and dominant  
* Avoid decorative fonts

---

### **Layout**

* Use grid systems  
* Maintain generous padding (at least 24–48px)  
* Avoid clutter

---

## **6\. Content Pipeline (AI-Assisted)**

### **Input**

Raw content (links, images, videos)

### **AI Tasks**

* Generate short editorial captions  
* Tag categories (architecture, fashion, objects)  
* Maintain consistent tone

---

### **Caption Style**

* Short (1–2 sentences)  
* Descriptive, not casual  
* No emojis or slang

Example:  
"Concrete minimalism defines this urban stadium, balancing mass with precision."

---

## **7\. Development Phases**

### **Phase 1 (MVP)**

* Static homepage  
* Content grid  
* Post pages  
* Basic navigation

---

### **Phase 2**

* Sidebar animation  
* Category filtering  
* Improved typography

---

### **Phase 3**

* CMS integration  
* TikTok content integration

---

## **8\. What to Avoid**

* Overengineering backend  
* Adding unnecessary animations  
* Using too many colors  
* Making it feel like a social app

---

## **9\. Definition of Done**

* Site loads fast  
* Design matches reference quality  
* Content feels curated  
* UI is clean and minimal
